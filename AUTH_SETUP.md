# Authentication Setup Guide with ngrok

## Quick Start (3 Steps)

### Step 1: Start ngrok Tunnel
```bash
npm run tunnel
```
**Wait for output like:**
```
🔗 PUBLIC NGROK URL: https://abc123.ngrok.io
```
**Copy this URL - you'll need it.**

### Step 2: Configure Supabase OAuth Settings
1. Go to https://app.supabase.com → Your Project → Authentication → Providers
2. For each provider (Google, etc.):
   - Add redirect URL: `https://YOUR_NGROK_URL/oauth-callback.html`
   - Example: `https://abc123.ngrok.io/oauth-callback.html`

### Step 3: Start App Server (in new terminal)
```bash
npm run dev
```
**Access at:**
- `https://YOUR_NGROK_URL/auth.html` (Login/Signup)
- `https://YOUR_NGROK_URL/dashboard.html` (Dashboard)

---

## How It Works

### ngrok Flow
1. **Local Server** (port 8000) → **ngrok Tunnel** → **Public HTTPS URL**
2. ngrok writes URL to `.env.local`
3. `config.js` reads `.env.local` and sets `window.BASE_URL`
4. All auth redirects use the ngrok URL

### Auth Flow
1. User clicks "Sign in with Google"
2. Redirects to Google OAuth endpoint
3. Google redirects back to: `https://your-ngrok-url/oauth-callback.html`
4. Supabase handles the callback
5. User redirected to dashboard

## Supabase Configuration

### Required OAuth Redirect URLs
Go to: **Authentication → Settings → Authorized redirect URLs**

Add ALL of these:
```
https://YOUR_NGROK_URL/oauth-callback.html
https://YOUR_NGROK_URL/dashboard.html
http://localhost:8000/oauth-callback.html
http://localhost:8000/dashboard.html
```

### Email Settings (Authentication → Providers)
- Confirm email: ON or OFF (your choice)
- If ON: Users get confirmation email before account activates
- If OFF: Instant account creation

## Troubleshooting

### "Invalid OAuth Redirect"
**Problem:** You're using `localhost:8000` but OAuth is configured for ngrok
**Solution:** 
1. Copy ngrok URL from terminal
2. Update Supabase redirect URLs with the ngrok URL
3. Access app via ngrok URL, not localhost

### "CORS Error"
**Problem:** App can't reach Supabase
**Solution:**
1. Check Supabase URL is correct in `config.js`
2. Ensure ngrok URL matches Supabase redirect URLs

### Auth Silent Fails
**Problem:** Click sign in, nothing happens
**Solution:**
1. Check browser console for errors (F12)
2. Verify ngrok tunnel is running (`npm run tunnel`)
3. Verify `.env.local` was created with NGROK_URL

### Session Not Persisting
**Problem:** Sign in works, but dashboard shows "not authenticated"
**Solution:**
1. Clear browser cache/cookies
2. Sign out completely
3. Sign in again
4. Check if email confirmation is blocking login

## Common Setup Mistakes

❌ **WRONG:**
- Using `http://localhost:8000` in Supabase OAuth settings
- Starting only `npm run dev` without ngrok
- Using old ngrok URL after restarting `npm run tunnel`

✅ **RIGHT:**
- Start `npm run tunnel` first
- Copy the new ngrok URL
- Update Supabase settings with the new URL
- Start `npm run dev` in second terminal
- Access via ngrok URL

## Resetting Everything

If something breaks:

```bash
# 1. Stop all processes (Ctrl+C in each terminal)

# 2. Remove old ngrok tunnel config
rm .env.local

# 3. Clear browser cache
# Chrome: Ctrl+Shift+Delete

# 4. Start fresh
npm run tunnel         # Terminal 1
npm run dev           # Terminal 2
```
